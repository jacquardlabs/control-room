| 1 | technical | Hook-based detection doesn't actually fire for tool calls made inside a Task-dispatched subagent — studious's own issue #97 hit exactly this ("dogfood item zero"): PostToolUse firing for the interactive session but not dispatched workers. If T1's hook path has the same blind spot, "faster than the harness" silently degrades to poll-only for most real usage, since most concurrent streams the owner cares about (epics, background fleets) *are* dispatched subagents, not the interactive session itself. | Check whether the T1 build states, and tests, which dispatch mode(s) the hook path actually fires in. An unstated or untested claim of "hook-first" here is the signal it hasn't been verified. |
| 2 | product | The R (review-ready) wall bucket is new UI surface with no comprehension test behind it (unlike grinding/need-you, which trace to the original Flight Deck test). The three-number wall could scan worse than the two-number one it replaced. | During issue #8's dogfood week, note any moment the owner misreads, ignores, or has to ask what the R count means. |
| 3 | product | Notification debounce ships without a real threshold (left as an open question) and a flickering detector produces the exact notification-fatigue the design warned about, burning trust before the dogfood week gets a fair read. | An incident-log entry during issue #8's week describing 3+ notifications for a stream that never left an ambiguous middle state. |
| 4 | technical | The hook-first/poll-fallback split means two code paths can independently judge one stream's attention state at different moments; if they briefly disagree (hook says input-blocked, next poll tick says grinding because its heuristic hasn't caught up), the wall could flicker between two truths for the same stream. | A build-time test driving both paths against the same fixture, asserting convergence — not just that each path independently degrades safely on its own. |
| 5 | product | The anti-false-amber invariant's adversarial fixtures are only as good as the ambiguous cases their author imagined; a real-world ambiguous transcript the fixture set didn't anticipate could still produce a false amber live, undetected until it burns trust. | Whether the fixture set had any review beyond its own author, or was purely self-authored and self-graded. |
| 6 | technical | The vendored cctx parsing copy has no recorded sync marker, so drift from cctx's own evolving format goes unnoticed until a stream silently fails to parse. | Check whether the vendored file carries any version/date/commit marker at all. |
